# YouTubeスケジュール — 運用ルール（三浦用）

**これだけ守れば、スマホとPCでデータがズレなくなる。**

---

## ブックマークは2つだけ

| どこで | ブックマークするURL |
|--------|---------------------|
| **PC（編集）** | https://takpz93.github.io/youtube-schedule/ |
| **スマホ（見るだけ）** | https://takpz93.github.io/youtube-schedule/?view=1&tab=week |

- スマホで `?view=1` **なし** のURLは使わない（古いデータの温床）
- `index.html` をローカルで開かない

---

## 編集したあと（PC）

**何もしなくてOK。** 編集すると **約1秒以内に自動でチーム・スマホへ反映** されます。

- 上部に **「☁️ チームへ自動反映中…」** と出たらそのまま待つ
- 右下に **「☁️ 自動反映済」** と時刻が出れば完了
- **赤いバナー（反映失敗）** だけ出たら再試行ボタンを押す（自動でも3回リトライします）
- 初回だけ **⚙️ 同期** でトークン設定が必要（未設定だと自動反映できない）

---

## スマホで見るとき

- **チームURL**（`?view=1`）だけ使う
- 上部に **「👥 チーム閲覧 — 15秒ごとに自動更新」** と出ていればOK
- 古いと感じたら **🔄 更新** を1回押す

---

## やってはいけないこと

| NG | 理由 |
|----|------|
| スマホで管理者URLをブックマーク | 端末内キャッシュとサーバーがズレる |
| 編集後にバナーを無視して閉じる | 失敗時のみ未反映の可能性（成功時は自動） |
| チームURLで編集しようとする | 編集不可。PCの管理者URLでやる |
| ローカルの `index.html` を開く | サーバーと別データ |

---

## トラブル時（30秒チェック）

1. **PC** … オレンジバナー → 「今すぐ公開」
2. **スマホ** … `?view=1` のURLか確認
3. **両方** … 右下のバージョンが同じか（例: `v20260702c`）
4. 直らなければ Safari でタブを閉じて開き直す

---

## 仕組み（覚えなくてOK）

```
PCで編集 → 約1秒で自動でGist/GitHubに反映
スマホ・チーム → サーバーから15秒ごとに取得
```

コードのデプロイは **もう案件データを上書きしない**（2026-07-02修正済み）。

---

## 初回だけ（済んでいれば不要）

1. PCで https://takpz93.github.io/youtube-schedule/ を開く
2. **⚙️ 同期** → ターミナル `gh auth token` のトークンを貼る
3. スマホのブックマークをチームURLに差し替え
